# AI Documenter Prompt Instructions

When writing documentation:
1. **Be concise:** Keep explanations short, clear, and focused.
2. **Align with implementation:** Avoid outdated guidelines. Update the `.agent/` operating manual when codebase behavior is refactored.
3. **Use Markdown:** Format using clean tables, headers, and codeblocks.
4. **Link references:** Use workspace relative schemes (e.g. `[filename](file:///path/to/file)`) to make documents clickable.
