# Coding Style Guidelines

- **Indentation:** Enforce 2-space indentation globally across CSS, JSON, TS, and TSX files.
- **Line Length:** Keep lines under 100-120 characters where practical.
- **Comments:** Retain existing file headers and docstrings. Document non-obvious architecture implementations clearly.
- **Semicolons:** Use semicolons consistently at the end of statements.
- **Formatting Checker:** Always verify changes by running `npm run check` which triggers Knip and ESLint analysis.
